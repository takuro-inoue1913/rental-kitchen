# 開発フロー

## Phase ごとの PR ルール
1. 各 Phase の実装が完了したらブランチを push し、PR を作成する
2. PR 作成後、GitHub Copilot にレビューをリクエストする
3. **PR が merge されるまで次の Phase に進まない**
4. レビュー指摘があれば **指摘1件につき1コミット** で修正する
5. 各指摘への返信は以下の構成で書く:
   - **理解**: レビュー指摘をどう理解したか
   - **解釈**: その問題がなぜ起きるか / どういうリスクがあるか
   - **対応**: 具体的にどう修正したか + commit URL
6. 全指摘への対応完了後、再度レビューを受ける
